Stage 2 : APO'G

Contexte

APO'G est une entreprise de services numériques, elle à réalisé une API pour un groupe scolaire qui permet de gérer les données des candidats souhaitant s'inscrire dans les écoles du groupe. Durant mon stage j'ai effectué diverses missions sur L'API qui m'ont permi de découvrir comment fonctionnent les API REST et comment les exploiter.

Mission 1 : Test et correction des points d'entrée de l'API

Mon maitre de stage m'a demandé de tester l'API avec le logiciel Postman, j'ai appris les bonnes pratiques permettant de rendre l'API restful et ai produit un premier compte rendu proposant une correction des points d'entrées.

  • Les bonnes pratiques REST
  • Diagnostic de l'API

Mission 2 : Documentation de l'API

On m'a ensuite demandé de créer une documentation de l'API avec la spécification OpenAPI, celle-ci permet de décrire une API REST en YAML ou en JSON et de générer une documentation dans divers formats à partir de ce code.

  • La page de documentation que j'ai créé pour l'API
  • Le code JSON permettant de générer la documentation

J'ai également réalisé une documentation au sujet de la spécification OpenAPI qui permettra à d'autres développeurs travaillant sur le projet de continuer la documentation.

  • La documentation explicant l'utilisation de la spécification OpenAPI

Mission 3 : Création d'une interface pour l'API

Enfin j'ai implémenté une interface permettant d'utiliser l'API (ajout ,supression et modification des données des candidats), pour cela j'ai appris à utiliser les frameworks Bootstrap et JQuery. J'ai réalisé plusieurs versions de l'interface afin de les adapter au demandes de l'utilisateur. Les données de l'API étaient affichées en AJAX (javascript asynchrone) afin de diminuer les temps de chargements qui était également une nouvelle technologies pour moi.

  • Première version de l'interface
  • Deuxième version de l'interface
  • Troisième version de l'interface
